# Pascual Reply Pro — Глобальный план: выжать максимум из x402/Arc

> Дата: 21.07.2026. Статус: v1.2.0 задеплоена (меню ✦, Analyze/Sentiment, x402-кредиты на Arc testnet, оплата проверена вживую).
> Цель документа: полная карта функций и улучшений, ранжированная по формуле **ценность для пользователя × демонстрация технологии × стоимость разработки**.

---

## 1. Стратегическая рамка

Продукт стоит на трёх ногах, и каждая новая функция должна усиливать хотя бы две:
1. **Полезность в X** — человек реально экономит время (ответы, анализ, контент-стратегия).
2. **x402/USDC-экономика** — каждая платная операция это микротранзакция без подписок и карт: живая демонстрация «programmable money».
3. **Агентность (Arc ERC-8004/8183)** — расширение постепенно превращается из «инструмента» в «агента, который зарабатывает и платит сам» — это главный приоритет грантов Circle.

Тарифная логика: **бесплатное = привлекает** (reply, базовые лимиты), **дешёвое по кредитам = удерживает** ($0.005–0.02 за операцию), **дорогое по кредитам = монетизирует** ($0.05–0.25 за отчёты/пакеты).

## 2. Функции — полный каталог

### Tier A — быстрые победы (дни, макс. эффект)

| # | Функция | Что даёт | Цена |
|---|---------|----------|------|
| A1 | **Improve my draft** — кнопка в композере: переписать мой черновик (3 варианта: короче/цепляюще/провокационнее) | Самая частая боль контент-мейкеров | 1 кредит |
| A2 | **История анализов** в попапе (уже логируются в `tweet_logs`) + просмотр/поиск | «Купил — вижу за что заплатил»; прозрачность трат | бесплатно |
| A3 | **Чек перед постом**: анализ моего готового твита ДО отправки — прогноз вовлечённости, риски (ratio, misread), совет по времени | Уникальная фича, никто не делает inline | 1 кредит |
| A4 | **Fact-check flag** — отдельный пункт меню: «проверь утверждения этого твита, что требует источника» | Растущий спрос, «белая» функция | 1 кредит |
| A5 | **Translate & explain** — перевод твита + культурный/сленговый контекст (у вас уже есть кодовая база translator'ов!) | Синергия портфеля Pascual Labs | бесплатно/1 |
| A6 | Отображение остатка кредитов в панели результата + тост «−1 кредит, осталось N» | Прозрачность микроплатежей = суть x402 | — |

### Tier B — продукт-дифференциаторы (1–2 недели)

| # | Функция | Что даёт |
|---|---------|----------|
| B1 | **Профиль-досье** (по кнопке на странице профиля): темы, тон, активность, «как заговорить с этим человеком», общие интересы | Киллер для нетворкинга/BD. Цена 3–5 кредитов — первый «дорогой» продукт |
| B2 | **Thread Composer** — из одной идеи собрать тред из N твитов с крючком и CTA | Высокая ценность, высокая цена (5 кредитов) |
| B3 | **Сравнение постов** — выделить 2+ своих твита → «почему этот зашёл, а тот нет» | Обучающая ценность, retention |
| B4 | **Watch-режим** (по кнопке, не фоном!): «следи за этим тредом, при открытии покажи что изменилось + новый сентимент» | Повторные сессии = daily active |
| B5 | **Пакеты кредитов** 50/$0.25, 250/$1, 1000/$3 + бонус за streak оплат | Классическая воронка + повод для повторных USDC-транзакций |
| B6 | **Реферальная механика в USDC**: пригласивший получает кредиты с первой покупки друга — выплата тоже ончейн | Вирусный рост, показывает P2P-выплаты |

### Tier C — агентный слой (2–4 недели; это ядро заявки на грант)

| # | Функция | Что даёт |
|---|---------|----------|
| C1 | **Регистрация worker'а как ERC-8004-агента** (скрипт `2_register_agent.py` уже есть!) — у сервиса появляется ончейн-идентичность и репутация | «Расширение обслуживается зарегистрированным Arc-агентом» — прямое попадание в нарратив Circle |
| C2 | **Каждый крупный анализ = ERC-8183 job**: создание job с deliverable = hash отчёта, complete после выдачи → **проверяемая ончейн-история оказанных услуг** | Превращает продукт в живой кейс Agentic Commerce; репутация = маркетинг |
| C3 | **API-витрина для чужих агентов**: тот же worker отдаёт /analyze как x402-endpoint для AI-агентов (не только людей) → листинг в agents.circle.com | Второй рынок сбыта без новой разработки; «агенты платят агентам» |
| C4 | **Агент-аналитик по подписке-стриму**: пользователь депонирует $1 в эскроу (8183), агент ежедневно готовит дайджест его ленты, списывая по факту | Идеальная демка programmable money: стриминговая оплата за работу |
| C5 | Привязка кредитов к **кошельку** (подпись-логин), а не к IP + история платежей в попапе со ссылками на arcscan | Убирает главный тех-долг; прозрачность = доверие |

## 3. Что сознательно НЕ делаем

- ❌ Авто-постинг, масс-реплаи, фоновый скрапинг ленты — бан по ToS X, репутационная смерть.
- ❌ Кошелёк/подпись внутри расширения — серая зона CWS; все платежи только на странице worker'а.
- ❌ Слово «airdrop/farming» в маркетинге — продукт продаёт пользу, ончейн-активность — побочный эффект.
- ❌ Свой токен. Только USDC.
